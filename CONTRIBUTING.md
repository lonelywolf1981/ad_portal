# Вклад в развитие проекта AD Portal

Спасибо за интерес к нашему проекту! Мы рады любому вкладу, будь то исправление ошибок, добавление новых функций или улучшение документации.

## Как внести вклад

### Сообщить об ошибке
1. Проверьте, не было ли уже создано issue с этой проблемой
2. Если нет, создайте новое issue с подробным описанием проблемы
3. Укажите версию Python, OS и шаги для воспроизведения

### Предложить улучшение
1. Создайте issue с описанием предлагаемого улучшения
2. Обсудите его с командой разработчиков
3. После одобрения приступайте к реализации

### Отправить pull request
1. Форкните репозиторий
2. Создайте ветку для новой функции (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Добавьте тесты, если применимо
5. Обновите документацию, если необходимо
6. Зафиксируйте изменения (`git commit -m 'Add some amazing feature'`)
7. Запушьте ветку (`git push origin feature/amazing-feature`)
8. Откройте Pull Request

## Настройка среды разработки

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/lonelywolf1981/ad_portal.git
   cd ad_portal
   ```

2. Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/Mac
   # или
   venv\Scripts\activate  # Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Скопируйте пример файла окружения:
   ```bash
   cp .env.example .env
   ```

5. Запустите приложение:
   ```bash
   docker-compose up --build
   ```

## Стиль кодирования

- Следуйте стандартам PEP 8 для Python кода
- Используйте типизацию везде, где возможно
- Пишите понятные имена переменных и функций
- Добавляйте docstrings к классам и функциям

## Тестирование

- Пишите unit-тесты для новых функций
- Убедитесь, что все существующие тесты проходят
- Проверьте, что ваш код работает в разных окружениях

## Структура проекта

- `app/` - основной код приложения
- `app/routers/` - маршруты FastAPI
- `app/services/` - бизнес-логика
- `app/templates/` - HTML-шаблоны
- `app/static/` - статические файлы
- `docker-compose.yml` - конфигурация Docker
- `nginx/` - конфигурация Nginx
